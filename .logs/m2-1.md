# M2.1 Log - Corrective Patch

Date: 2026-05-16

## Issues

- Manual playtest found the placeholder Incident Report visually intersecting room geometry.
- Manual playtest found that the player could walk through walls and objects.

## Cause

- M2 set Babylon `checkCollisions` flags on procedural meshes, but the player controller moved by directly mutating `camera.position`, bypassing Babylon's built-in collision movement.
- The report was positioned near the cubicle doorway/partition area, which made it appear embedded in geometry from normal play angles.

## Fix

- Added `src/game/navigation/collision.ts`, a deterministic AABB movement collision system derived from source-created collidable meshes.
- Updated `src/game/player.ts` to resolve requested movement through the collision system while preserving speed, camera, mouse look, and restart feel.
- Moved the placeholder report to the lobby reception desk and disabled back-face culling on the report material so the paper remains visible.
- Added debug proof for collision blocker count, a blocked wall probe, five clear doorway probes, and corrected report position.
- Updated smoke proof to save `.logs/m2-1-smoke.json`, `.logs/m2-1-smoke.png`, and `.logs/m2-1-report.png`.
